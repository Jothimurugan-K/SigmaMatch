🐳 Self-Host SigmaMatch with Docker

Run this tool entirely inside your own network. Logs never leave your infrastructure.

Prerequisites

  • Docker installed
  • Git installed

Step 1 — Clone the Repository

git clone https://github.com/Jothimurugan-K/SigmaMatch.git cd SigmaMatch

Step 2 — Build & Run

docker-compose up -d --build

This builds the image, installs dependencies, and starts the server on port 8000.

Step 3 — Open the Tool

http://localhost:8000

That’s it — the tool is running entirely on your machine. Now you can safely use real logs.

Useful Commands

ActionCommand
View logsdocker-compose logs -f
Stopdocker-compose down
Restartdocker-compose restart

Optional Configuration

Add these under environment: in docker-compose.yml:

VariableDefaultPurpose
ALLOWED_ORIGINSsame-originCORS allowed origins (comma-separated)
MATCH_TIMEOUT30Max seconds per match operation

Change Port

If port 8000 is taken, edit docker-compose.yml and change "8000:8000" to e.g. "9090:8000", then access at http://localhost:9090.
